## 一、系统总览

有膜有漾 CRM 是一套覆盖从**获客到售后全链路**的飞书多维表格系统，包含 7 张数据表，通过关联字段打通客户旅程的每个环节。

### 整体架构

```
线索管理 ──→ 跟进记录 ──→ 报价表 ──→ 客户车辆档案 ──→ 工单表 ──→ 售后表
    │            │            │            │             │           │
    │            │            │            │             │           │
    └──── 销售人员表 ──── 产品项目表 ──── 施工师傅表 ────┘
```

### 核心数据流向

```
获客 → 跟进 → 报价 → 成交 → 建档 → 施工 → 售后
 ①      ②      ③      ④      ⑤      ⑥      ⑦
```

## 五、全链路状态流转

```
线索状态：未分配 → 已分配 → 已转商机 → 无效线索

报价状态：草稿 → 已发客户 → 跟进中 → 已成交 / 已作废

工单状态：待安排 → 待施工 → 施工中 → 待验收 → 待交付 → 已关闭
                                                          ↘ 已取消

售后状态：待处理 → 处理中 → 已解决 → 已关闭

客户状态：潜在客户 → 合作客户 → 忠诚客户 → 流失客户
```

## 六、常见问题

**Q：为什么有些字段要填两个名字？**
A：线索管理的「客户姓名」和「姓名」是两个独立的文本字段，写入时都需要填相同值。

**Q：车型字段为什么只能选 SUV/MPV/轿车？**
A：车型字段是单选类型，具体车型名（如问界M9）请写入「车辆品牌」或「线索描述」。

**Q：销售负责人用 open_id 还是 record_id？**
A：看字段类型。线索管理/跟进记录/报价表/工单表的「销售负责人」是**关联字段**，用 `record_id`。跟进记录的「跟进人」是**人员字段**，用 `open_id`。

**Q：电动踏板属于什么产品类型？**
A：在「意向产品类型」中映射为「轻改装」，在「产品项目表」中有独立类型「电动踏板」。
