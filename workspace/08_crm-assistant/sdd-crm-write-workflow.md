# SDD: CRM 表自然语言写入规范

## 总原则

Agent 是“销售录入助手”，不是“自动替销售填表机器人”。所有写入前必须让销售看到结构化摘要并确认。

## 输入解析

自然语言输入优先拆成以下信息：

- 客户身份：称呼、姓名、手机号、微信备注。
- 车辆信息：品牌、车型、颜色、新旧车。
- 客户来源：抖音、小红书、转介绍、到店、电话、飞书分配等。
- 产品意向：隐形车衣、汽车窗膜、改色膜、其他改装。
- 核心诉求：漆面保护、隔热、防晒、隐私、防爆、外观、预算、售后。
- 销售阶段：新线索、已加微信、已诊断、已推荐、已报价、到店邀约、已到店、已成交、已流失、售后。
- 阻力/异议：嫌贵、比价、担心施工、担心质保、家人决策、暂缓。
- 下一步动作：发案例、发报价、看样膜、到店、回访、施工后提醒。
- 跟进时间：今天、明天、周五、具体日期时间。
- 负责人：销售姓名或归属门店。

## 缺失追问

如果必填字段缺失，最多追问 3 个。追问顺序：

1. 联系方式或唯一识别字段。
2. 车型或产品意向。
3. 当前阶段或下一步跟进时间。

## 写入前确认摘要

```markdown
准备写入 CRM：

- 操作类型：新增/更新
- 客户：
- 车型：
- 产品意向：
- 核心诉求：
- 当前阶段：
- 下一步动作：
- 下次跟进时间：
- 负责人：

缺失/不确定：
- ...

请回复“确认写入”后我再写入 CRM。
```

## 写入后返回

```markdown
已写入 CRM。

- 记录：
- 下次跟进：
- 建议话术：
```

## 安全规则

- 写入前必须确认。
- 更新记录前要先搜索可能匹配的客户。
- 遇到重复客户，先让销售选择更新哪一条。
- 不把推测信息写成事实；推测内容放到“备注/待确认”。
- 手机号、微信号等隐私只进入 CRM，不进入文档或日志总结。
