# CRM 表结构建议

当前 Base `多维表格 CRM 表设计` 已可访问，但当前账号读取到的数据表数量为 0。因此这里先定义适合有膜有漾销售流程的 CRM 表设计，后续可以按此在飞书 Base 中创建真实表。

## 表 1: 客户线索表

这是销售每天主要填写和查看的主表。自然语言录入优先写入这张表。

| 字段名 | 类型建议 | 必填 | 用途 |
| --- | --- | --- | --- |
| 客户名称/称呼 | 文本 | 是 | 销售识别客户 |
| 联系方式 | 电话/文本 | 是 | 查重和后续联系 |
| 微信备注 | 文本 | 否 | 没有手机号时辅助识别 |
| 来源渠道 | 单选 | 是 | 抖音、小红书、转介绍、到店、电话、老客复购等 |
| 归属门店 | 单选/关联 | 是 | 区分门店 |
| 负责人 | 人员 | 是 | 负责跟进的销售 |
| 车型 | 文本 | 是 | 推荐产品的关键依据 |
| 车辆状态 | 单选 | 否 | 新车、准新车、已使用、已施工 |
| 产品意向 | 多选 | 是 | 隐形车衣、汽车窗膜、改色膜、其他 |
| 核心诉求 | 多选 | 否 | 漆面保护、隔热、防晒、隐私、防爆、外观、预算、售后 |
| 预算范围 | 单选/文本 | 否 | 低/中/高或客户原话 |
| 当前阶段 | 单选 | 是 | 新线索、已加微信、已诊断、已推荐、已报价、到店邀约、已到店、已成交、已流失、售后 |
| 客户异议 | 多选 | 否 | 价格、施工风险、质保、竞品比价、家人决策、暂缓 |
| 下一步动作 | 文本/单选 | 是 | 发案例、发报价、看样膜、到店、回访、施工后提醒 |
| 下次跟进时间 | 日期时间 | 是 | 推动销售动作 |
| 最近跟进摘要 | 长文本 | 否 | 最新一次沟通摘要 |
| 流失原因 | 单选/文本 | 条件必填 | 当前阶段为已流失时必填 |
| 成交产品 | 文本/关联 | 条件必填 | 当前阶段为已成交时必填 |
| 备注 | 长文本 | 否 | 客户原话或特殊情况 |

## 表 2: 跟进记录表

用于保留客户每一次沟通记录，避免主表备注越来越乱。

| 字段名 | 类型建议 | 必填 | 用途 |
| --- | --- | --- | --- |
| 关联客户 | 关联 | 是 | 关联客户线索表 |
| 跟进时间 | 日期时间 | 是 | 本次沟通发生时间 |
| 跟进方式 | 单选 | 是 | 微信、电话、到店、飞书、短信 |
| 沟通摘要 | 长文本 | 是 | 销售自然语言摘要 |
| 客户反馈 | 长文本 | 否 | 客户原话 |
| 本次异议 | 多选 | 否 | 价格、施工、质保、竞品、决策链等 |
| 推荐方案 | 文本 | 否 | 本次推荐的产品/套餐 |
| 下一步动作 | 文本 | 是 | 后续要做什么 |
| 下次跟进时间 | 日期时间 | 否 | 如果需要回访则填写 |
| 记录人 | 人员 | 是 | 谁记录的 |

## 表 3: 产品意向表

用于沉淀产品方向、推荐理由和标准话术，也可先不建，直接使用现有知识库和 skills。

| 字段名 | 类型建议 | 必填 | 用途 |
| --- | --- | --- | --- |
| 产品方向 | 单选 | 是 | 隐形车衣、汽车窗膜、改色膜 |
| 产品/套餐名称 | 文本 | 是 | 例如春分套餐、YM-60 |
| 适合客户 | 长文本 | 否 | 场景说明 |
| 推荐理由 | 长文本 | 否 | 销售解释 |
| 来源页 | 文本 | 否 | 知识库来源 |

## 表 4: 门店政策表

价格、活动、库存、排期不进入长期知识库，但可以进入带有效期的门店政策表。

| 字段名 | 类型建议 | 必填 | 用途 |
| --- | --- | --- | --- |
| 政策类型 | 单选 | 是 | 价格、活动、库存、排期 |
| 适用门店 | 单选/关联 | 是 | 门店范围 |
| 政策内容 | 长文本 | 是 | 具体说明 |
| 生效时间 | 日期时间 | 是 | 开始时间 |
| 失效时间 | 日期时间 | 是 | 结束时间 |
| 负责人 | 人员 | 是 | 谁维护 |
| 状态 | 单选 | 是 | 生效中、已失效、待确认 |

## Agent 填写优先级

1. 新客户或客户信息变化：写 `客户线索表`。
2. 每次沟通事实：写 `跟进记录表`。
3. 推荐方案：优先查现有 `knowledge-base/` 和 `skills/`，必要时同步到 `产品意向表`。
4. 报价、活动、排期：只引用 `门店政策表` 中生效且匹配门店的记录。
