# SDD: 人与 AI 协作规范

## 规范目的

本规范定义销售人员、运营人员、知识库维护者和 Hermes agent 之间的协作规则，减少口径混乱、过度承诺和不可追溯回答。

## 输入规范

销售向 agent 提问时，优先提供：

- 客户阶段：首次咨询、已看套餐、已报价、比价中、施工后、售后争议。
- 车型：品牌、车型、是否新车。
- 客户诉求：保护、隔热、隐私、防爆、外观、预算、售后。
- 当前阻力：嫌贵、犹豫、比较竞品、担心施工、担心质保。
- 输出目标：诊断问题、推荐方案、客户话术、跟进消息、FAQ 回答、改色效果图。

## 输出规范

Agent 输出应优先包含：

- 直接结论：先说建议或判断。
- 可复制话术：给销售能直接发客户的版本。
- 推荐依据：说明为什么匹配客户场景。
- 下一步动作：确认车型、看样膜、对比套餐、到店检查、预约施工。
- 边界提醒：价格、活动、库存、排期以门店最新确认为准。

## 强制边界

- 不编造价格、折扣、库存、施工排期。
- 不承诺手册或门店政策没有确认的质保和售后责任。
- 不攻击竞品，不制造虚假稀缺。
- 不把 AI 改色图包装成真实施工承诺。
- 参数、质保、售后类回答需要来源页时，必须引用知识库来源。

## 变更流程

1. 销售或客服在 `workspace/06_operations/sales-feedback-intake.md` 记录问题。
2. 运营判断是话术问题、知识缺口、技能路由问题还是门店政策问题。
3. 知识库维护者更新 `knowledge-base/` 或补充 workspace 规范。
4. 在 `workspace/05_evaluation/regression-log.md` 增加验收问题。
5. 通过后记录到 `workspace/06_operations/changelog.md`。
