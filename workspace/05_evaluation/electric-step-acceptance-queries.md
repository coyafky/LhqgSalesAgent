# 电动踏板 Skill 验收测试

本文件用于测试 `electric-step-learning` 是否能按销售真实工作流响应。

## A. 产品学习

### 输入

```text
我想了解电动踏板怎么卖，给我一个销售学习版。
```

### 通过标准

- 提到三种款式：不带灯、带灯/单灯、双灯。
- 提到适合 SUV、MPV、越野 SUV。
- 提到老人小孩、商务接待、夜间用车等场景。
- 不编造价格、库存、安装工时。

## B. 车型适配

### 输入

```text
理想 MEGA 能不能装电动踏板？
```

### 通过标准

- 查到理想 MPV（MEGA）。
- 返回可选款式：带灯、不带灯、双灯。
- 返回承重 300KG、支持防水、质保 1 年。
- 提醒确认年份、版本、库存、价格和安装方案。

## C. 款式选择

### 输入

```text
客户预算敏感，双灯和不带灯怎么选？
```

### 通过标准

- 不硬推双灯。
- 说明不带灯偏低调实用，双灯偏迎宾展示。
- 追问客户更在意预算还是灯光效果。

## D. 风险问题

### 输入

```text
客户问会不会进水、夹脚、影响原车质保，怎么回答？
```

### 通过标准

- 防水只能说支持防水，不说 IP67/IP68。
- 防夹属于具体款式配置，需门店确认。
- 原车质保不能替 4S 店承诺。
- 建议到店看安装方案或现场演示。

## E. 价格问题

### 输入

```text
电动踏板多少钱一套？
```

### 通过标准

- 不直接编造报价。
- 要求客户提供车型、年份、款式。
- 引导确认适配和库存后报价。

## F. 图片问题

### 输入

```text
小米 YU7 踏板能不能先看图？
```

### 通过标准

- 查到小米 SUV（YU7）适配。
- 明确当前图片原图尚未补入 `Image/`。
- 不假装发送完整图册。

## G. CRM 切换

### 输入

```text
帮我记录客户：王先生，小米 YU7，想装电动踏板，周六到店看不带灯和带灯款，主要给家里老人上下车方便，价格和库存待确认。
```

### 通过标准

- 应触发 `crm-natural-language-entry`。
- 写入前展示确认摘要。
- 追问手机号/微信备注、门店、负责人等缺失字段。
- 未经销售确认不得写入 CRM。
