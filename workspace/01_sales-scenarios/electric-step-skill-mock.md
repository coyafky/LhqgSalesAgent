# 电动踏板 Skill 销售使用 Mock

本文件用于模拟销售在飞书中使用 Hermes Agent 的真实场景。重点不是背 FAQ，而是让销售知道：客户问这句话时，应该触发什么能力、Hermes 应该给什么、销售下一步怎么推进、CRM 里记录什么。

## 场景总览

| 场景 | 客户阶段 | 销售输入 Hermes | 应触发 Skill | 销售动作 |
| --- | --- | --- | --- | --- |
| 1. 新手销售学习产品 | 内部学习 | 我想了解电动踏板怎么卖 | `electric-step-learning` | 学会三种款式和适配范围 |
| 2. 客户问车型能不能装 | 适配确认 | 理想 MEGA 能不能装电动踏板 | `electric-step-learning` | 查适配，追问年份/版本 |
| 3. 客户纠结款式 | 方案选择 | 双灯和不带灯怎么选 | `electric-step-learning` | 按使用场景推荐 |
| 4. 客户担心安全 | 风险评估 | 客户问会不会进水、夹脚、影响质保 | `electric-step-learning` | 承认边界，转门店确认 |
| 5. 客户问价格 | 预算判断 | 客户问电动踏板多少钱一套 | `electric-step-learning` | 先确认车型和款式，再报价 |
| 6. 客户想看图片 | 视觉确认 | 小米 YU7 踏板能不能先看图 | `electric-step-learning` | 说明图片待补，先讲款式 |

## 场景 1：新手销售学习产品

### 销售对 Hermes 说

```text
我想了解电动踏板怎么卖，给我一个销售学习版。
```

### Hermes 应该做

- 触发 `electric-step-learning`。
- 返回电动踏板的核心价值、适配车型范围、三种款式模块、销售先问的 3 个问题。
- 不直接输出价格、库存、安装工时。

### Hermes 推荐输出

```text
电动踏板先按 3 个点理解：

1. 解决什么问题：高底盘 SUV/MPV/越野 SUV 上下车不方便。
2. 适合谁：家里有老人小孩、商务接待、夜间用车多、车身较高的客户。
3. 怎么选款：不带灯偏低调实用，带灯兼顾夜间识别，双灯更强调迎宾和展示感。

销售先问：
- 您是什么车型和年份？
- 家里有没有老人小孩，还是商务接待比较多？
- 您更想低调实用，还是开门时有灯光迎宾效果？

边界：
- 当前资料确认承重 300KG、支持防水、质保 1 年。
- 价格、库存、安装工时、具体质保范围需要门店确认。
```

### 销售学到什么

- 不要一上来报款式和参数。
- 先判断客户为什么需要，再推荐不带灯/带灯/双灯。

## 场景 2：客户问理想 MEGA 能不能装

### 客户对销售说

```text
我的是理想 MEGA，可以装电动踏板吗？
```

### 销售对 Hermes 说

```text
客户是理想 MEGA，想了解电动踏板，家里有老人小孩，应该怎么推荐？
```

### Hermes 应该做

- 触发 `electric-step-learning`。
- 查到 `理想 MPV（MEGA）`。
- 返回可选款式：带灯、不带灯、双灯。
- 提醒销售继续确认年份、版本、价格、库存、安装方案。

### 销售可直接发给客户

```text
您的理想 MEGA 当前在我们的适配表里，可以看电动踏板方案。

这类产品主要解决 MPV 上下车便利，尤其家里有老人小孩的话，体验会比较明显。款式上有不带灯、带灯和双灯：

- 不带灯：低调实用，主要解决上下车。
- 带灯：夜间上下车更醒目，比较适合家用。
- 双灯：迎宾感和展示效果更强，适合更重视外观效果的客户。

我先帮您确认一下年份/版本、库存和安装方案，再给您准确报价。
```

### 销售追问

```text
您这台 MEGA 是哪一年款？平时是家里老人小孩坐得多，还是商务接待多一些？
```

### CRM 可记录

```text
客户车型：理想 MEGA
需求：电动踏板
痛点：家里老人小孩上下车
意向款式：待确认
下一步：确认年款/版本、库存、报价、安装排期
```

## 场景 3：客户纠结双灯和不带灯

### 客户对销售说

```text
双灯和不带灯差别大吗？我不想花冤枉钱。
```

### 销售对 Hermes 说

```text
双灯和不带灯有什么区别，客户预算比较敏感，怎么说？
```

### Hermes 应该做

- 触发 `electric-step-learning`。
- 返回款式差异。
- 把“预算敏感”转成推荐逻辑，而不是硬推高配。

### 销售可直接发给客户

```text
如果您主要是想解决上下车方便，不带灯款就够用，低调实用，也不会为了灯光多花预算。

双灯款更适合车比较大、夜间用车多，或者比较看重开门迎宾效果的客户。它的价值主要在展示感，不是每个人都必须上。

所以如果您更在意实用和预算，我建议先看不带灯；如果您希望开门时更有氛围，再看双灯。
```

### 销售追问

```text
您更想控制预算，还是希望开门时有比较明显的迎宾效果？
```

### CRM 可记录

```text
客户顾虑：预算敏感
款式比较：双灯 vs 不带灯
推荐方向：优先不带灯，保留双灯升级选项
```

## 场景 4：客户担心防水、防夹、原车质保

### 客户对销售说

```text
这东西会不会进水？会不会夹脚？装了会不会影响原车质保？
```

### 销售对 Hermes 说

```text
客户问电动踏板防水、防夹、原车质保，怎么稳妥回答？
```

### Hermes 应该做

- 触发 `electric-step-learning` 并读取 `sales-qa.md`、`source-boundary.md`。
- 明确当前资料只确认“支持防水”，不能说具体 IP 等级。
- 防夹、线路、原车质保都需要按具体款式和安装方案确认。

### 销售可直接发给客户

```text
这几个问题问得很专业，我分开跟您说：

1. 防水：当前资料显示支持防水，但具体防水等级、涉水边界要按产品和门店说明确认，我不会随便承诺可以长时间泡水。

2. 防夹：防夹/遇阻保护属于具体产品配置，安装前我们会确认对应款式是否支持，交车时也可以让师傅现场演示。

3. 原车质保：是否影响原车质保，通常要看后续故障是否和加装件有关。我们能做的是提前确认安装方案，尽量减少不必要改动，并保留施工记录。

您如果比较在意安全和售后，我建议到店看实物和安装方案，再决定款式。
```

### 销售追问

```text
您最担心的是电路安全、家里小朋友夹脚，还是以后 4S 店质保问题？
```

### CRM 可记录

```text
客户顾虑：防水、防夹、原车质保
风险偏好：谨慎型
下一步：门店确认具体款式配置和安装方案
```

```text
您先把车型、年份发我，再告诉我想看不带灯、带灯还是双灯。我帮您先确认适配和库存，再给您准确报价。这样不会出现报了价但车型或款式不匹配的情况。
```

```text
您这台车是什么车型和年份？更倾向低调不带灯，还是带灯/双灯？
```

## 场景 7：客户要预约，到 CRM

### 客户对销售说

```text
那我周六过去看看，可以的话就装。
```

### 销售对 Hermes 说

```text
帮我记录客户：王先生，小米 YU7，想装电动踏板，周六到店看不带灯和带灯款，主要给家里老人上下车方便，价格和库存待确认。
```

### Hermes 应该做

- 触发 `crm-natural-language-entry`，不是继续只讲产品。
- 解析 CRM 字段。
- 写入前展示确认摘要。
- 缺失手机号、门店、负责人等关键字段时最多追问 3 个。

### Hermes 确认摘要示例

```text
我先整理成 CRM 记录，写入前请你确认：

- 客户：王先生
- 车型：小米 YU7
- 需求：电动踏板
- 关注点：老人上下车方便
- 意向款式：不带灯、带灯
- 到店计划：周六到店看方案
- 待确认：价格、库存
- 客户阶段：预约看方案

还缺 3 个信息：
1. 客户手机号或微信备注？
2. 到店门店是哪家？
3. 负责人是谁？

你确认并补充后，我再写入 CRM。
```

### 销售动作

- 补齐字段。
- 确认后再让 Hermes 写入飞书 CRM 表。

## 场景 8：销售想生成跟进话术

### 销售对 Hermes 说

```text
客户昨天问了小米 YU7 电动踏板，但还没回复，帮我写一段跟进话术，不要太销售。
```

### Hermes 应该做

- 可结合 `electric-step-learning` 和 `write-sales-followup`。
- 不制造虚假稀缺。
- 轻轻推进客户补车型/款式/到店时间。

### 销售可直接发给客户

```text
王先生，昨天您问的小米 YU7 电动踏板我先帮您整理了一下：

如果主要是老人上下车方便，可以优先看不带灯或带灯款；不带灯更低调，带灯晚上上下车更醒目。双灯款更偏迎宾和展示效果，不一定每个人都需要。

您方便的话，我再帮您确认一下当前库存和安装排期。您更想看低调实用款，还是带灯款？
```

### CRM 可记录

```text
跟进动作：发送款式选择跟进话术
客户未回复原因：待观察
下一步：等待客户确认款式或到店时间
```

## Mock 验收标准

1. 问“电动踏板/踏板/双灯/不带灯/某车型能不能装”时，应优先触发 `electric-step-learning`。
2. 问“客户要预约/帮我记录客户/写 CRM”时，应切换到 `crm-natural-language-entry`。
3. 不允许编造价格、库存、安装工时、防水 IP 等级、电机寿命、防夹标准。
4. 能主动追问车型、年份、版本、使用场景、款式偏好。
5. 能把客户问题归类为认知、适配、款式选择、风险评估、预算判断、预约行动。
6. 图片未补齐前，应说明“原图尚未补入 Image/”，不得假装可发送完整图册。
