# CRM售后助手

## 概述

客户施工后反馈问题或做售后回访时，销售一句话记录售后工单：搜索客户档案和原工单，填写问题描述和售后类型，写入「售后表」。

## 工作流程

### 第一步：搜索客户

在**客户车辆档案表**中搜索客户：`+record-search --search-field fld1lpoKfI --keyword "王" --as bot`

获取客户的：
- 档案 record_id
- 最近的工单 record_id（通过关联查询）

### 第二步：搜索原工单

在**工单表**中搜索该客户已关闭的工单，获取原工单 record_id。

### 第三步：解析售后信息

- 售后类型（返工/质保/投诉/咨询/保养/复购回访）
- 问题描述
- 售后状态
- 预约处理时间

### 第四步：追问缺失（最多3个）

1. 什么问题？（问题描述）
2. 售后类型（返工还是质保？）
3. 预约什么时候处理？

### 第五步：确认摘要

```markdown
📋 准备写入 CRM — 售后

| 字段 | 值 |
|------|-----|
| 客户 | XXX（关联档案：recXXX） |
| 原工单 | XXX（工单编号） |
| 售后类型 | XXX |
| 问题描述 | XXX |
| 售后状态 | XXX |
| 负责销售 | XXX |

请回复「确认写入」后我写入 CRM。
```

### 第六步：写入

先按 `../references/idempotency.md` 判断是新建售后问题还是更新已有售后状态。解决、关闭、满意度更新必须绑定明确售后 `record_id` 或经用户从候选列表选择。

新建售后记录：

```bash
lark-cli base +record-upsert \
  --base-token Q6VQbuLTOaPP5VspfKXc7ZbTnA8 \
  --table-id tbliYr7rZow5cBIq \
  --json '{
    "客户":[{"id":"recvmr4pCUVK2x"}],
    "原工单":[{"id":"recvmr4rTdFwzW"}],
    "售后类型":"返工",
    "问题描述":"车衣边缘有轻微翘起，约2cm",
    "售后状态":"待处理",
    "负责销售":[{"id":"rec6gBu3IVdtim"}],
    "预约处理时间":"2026-06-18 00:00:00"
  }' \
  --as bot
```

更新售后状态：

```bash
lark-cli base +record-upsert \
  --base-token Q6VQbuLTOaPP5VspfKXc7ZbTnA8 \
  --table-id tbliYr7rZow5cBIq \
  --record-id <已有售后ID> \
  --json '{
    "售后状态":"已解决",
    "处理方案":"重新热风收边处理，已修正",
    "完成时间":"2026-06-18 00:00:00",
    "客户满意度":"满意"
  }' \
  --as bot
```

## 安全规则

- 遵守 `../references/base.md` 的确认边界和隐私规则。
- 遵守 `../references/idempotency.md`：售后状态更新必须定位到明确售后记录。
- 写入前必须找到对应的客户档案和原工单，无法确认时先让销售指定。
- 没有销售确认，不直接写入。
- 不凭空编造问题描述和处理方案。
- 客户满意度以客户实际反馈为准，不代客户判断。
