# CRM数据驱动话术生成

## 概述

销售说"给XX客户升单"或"我要跟进XX"，助理自动：

1. **查CRM** — 获取线索信息 + 跟进记录 + 报价/档案/工单历史
2. **建画像** — 车型、已购/意向产品、预算、顾虑、历史沟通
3. **生成升单话术** — 按客户所处阶段输出可直接复制发给客户的版本

## 触发条件

| 销售说 | 识别关键词 |
|--------|-----------|
| *给XXX升单 / 加一些其他服务* | **升单/增项/加项目** |
| *XXX这个客户怎么跟进 / 有什么机会* | **跟进机会/怎么聊** |
| *给XXX推荐点别的 / 还能做什么* | **推荐/还能做什么** |
| *XXX成交了，还能推什么* | **已成交+追加** |
| *XXX这个客户怎么聊 / 怎么跟进* | **跟进引导/话术** |

## 工作流程

### 第一段：CRM数据查询

#### 1.1 线索定位

**⚠️ 前置步骤：先查字段ID**

`+record-search` 的 `--search-field` 参数需要传**字段ID**而不是字段名。首次使用或不确定字段ID时，先执行：

```bash
lark-cli base +field-list \
  --base-token Q6VQbuLTOaPP5VspfKXc7ZbTnA8 \
  --table-id tblHsrpSsTmJnB0c \
  --as bot --format compact
```

从结果中找到「客户姓名」字段的ID（当前为 `fldC6n9VPy`）。

**搜索线索：**

```bash
lark-cli base +record-search \
  --base-token Q6VQbuLTOaPP5VspfKXc7ZbTnA8 \
  --table-id tblHsrpSsTmJnB0c \
  --search-field fldC6n9VPy \
  --keyword "<客户名>" \
  --as bot --limit 10
```

**⚠️ 重要：同名客户归属判断**

搜索结果可能包含多条同名客户，分属不同销售。必须用 **销售负责人字段（fldILyNJFZ）** 筛选归属：

- 结果中 `销售负责人` 列会显示 `[{"id":"recXXX"}]`，对照以下映射表判断归属
- 如果当前销售是全名（如"吕佳豪"），客户是尊称（如"王先生"），两者不同名但仍是同一个人

**销售负责人 record_id 映射表：**

| 销售名 | 销售人员表 record_id |
|--------|---------------------|
| 吕佳豪 | `rec6gBu3IVdtim` |
| 张秋苑 | `reciu97dvVdCDK` |
| 刘志培 | `recERZnykdciRR` |

**找不到匹配的归属时**（搜索结果中销售负责人为空或指向其他销售），让销售确认具体是哪一条线索。

**补充：搜索结果中无跟进记录**

如果搜索结果中 `跟进记录` 列为空，说明尚未有任何跟进。直接用线索描述中的车型、意向产品、来源渠道构建话术。

| 车型 | fld579cOgh | SUV/MPV/轿车 分类 |
| 意向产品类型 | fld5ec6aAM | 当前关注点 |
| 线索描述 | fldEbfKitb | 过往沟通细节 |
| 线索状态 | fldXQfPxx7 | 决策阶段判断 |
| 来源渠道 | flddSSNiF8 | 来源信任度 |
| 预算范围 | fldwZfsV6Q | 价格敏感度 |
| 客户标签 | fldrMHyU9N | 客户价值评级 |
| 销售负责人 | fldILyNJFZ | 确认归属 |

#### 1.2 跟进记录查询

```bash
lark-cli base +record-get \
  --base-token Q6VQbuLTOaPP5VspfKXc7ZbTnA8 \
  --table-id tblHsrpSsTmJnB0c \
  --record-id <线索record_id> \
  --as bot
```

跟进记录字段是 link 类型（fldBG0DbC7），关联到跟进记录表。通过跟进记录可以获取：
- 客户说过的原话（异议、顾虑、需求）
- 之前推荐的方案
- 上次跟进时间和结论

#### 1.3 报价/档案/工单查询（如适用）

如果客户已成交，还要查：
- 报价记录：买过什么产品、价格区间
- 客户档案：车辆具体配置（年款、颜色）
- 工单状态：是否已施工、施工项目

### 第二段：客户画像分析

从CRM数据中构建五维画像：

| 维度 | 分析要素 | 升单方向 |
|------|---------|---------|
| 🚗 **车型画像** | 品牌+SUV/MPV/轿车 → 推断消费力 | 高消费力车型优先推车衣/改色 |
| 🛍️ **购买阶段** | 未购/已购/已施工 | 未购的推套装，已购的推互补品 |
| 💰 **价格敏感度** | 预算范围+砍价历史 | 高敏感度强调性价比，低敏感度推高端 |
| 🤔 **顾虑点** | 跟进记录中的异议 | 核心顾虑优先回应再推 |
| 🔗 **产品关联** | 已购/意向产品 → 关联互补品 | 车衣关联窗膜，踏板关联车衣 |

**产品关联升单矩阵（标准参照）：**

| 已有/意向 | 自然升单方向 | 理由 |
|----------|-------------|------|
| 电动踏板 | 隐形车衣 / 窗膜 | 已有基础轻改装需求 → 来都来了，一次到位 |
| 隐形车衣 | 窗膜 / 改色 | 新车全保护，一站式省心 |
| 窗膜 | 隐形车衣 / 精洗养护 | 隔热已经安排了，漆面保护也要跟上 |
| 改色膜 | 隐形车衣（先上车衣再改色） | 改色后原车漆更需要保护 |
| 精洗养护 | 窗膜 / 隐形车衣 | 已有保养意识 → 升级到长期保护 |
| 已购套餐 | 次年到店复检时追加 | 施工后一年复检时推+精洗套餐 |

### 第三段：话术生成规则

#### 3.1 话术结构（三段式）

```
[切入] — 从客户已有的购买/意向切入，自然过渡
[价值] — 新推产品的具体价值（针对该车型）
[下一步] — 具体的邀约到店或线上确认
```

#### 3.2 不同阶段的升单策略

| 客户阶段 | 策略 | 示例 |
|---------|------|------|
| 新线索·未回复 | 首次联系，先用入口产品建立信任 | "王先生，您问的电动踏板..." |
| 已沟通·未到店 | 到店邀约 + 自然带出多项目 | "踏板有不同款式，您到店看看，顺便对比下车衣效果" |
| 已到店·未定 | 现场对比推动决策 | "您看这个膜装上的效果，M9这么大面积..." |
| 已成交·已施工 | 施工后回访+自然追加 | "您的车衣用着怎么样？下次有空回来做个精洗养护" |
| 已转商机·流失 | 唤醒引入新产品 | "好久没联系，我们新到了一个改装件..." |

#### 3.3 话术风格规范

- 每段话术**可以直接复制发给客户**
- 避免绝对化表达（"最好""第一""唯一"）
- 不贬低竞品，用事实对比
- 价格留门店确认，不给具体数字承诺
- 不编造热销、限时、名额等紧迫感

## 安全规则

- 话术中的价格、优惠、活动、存库信息必须标注"以门店确认为准"
- 不编造客户案例或热销数据
- 不把AI分析的"可能"当成客户事实
- 车型升单建议只做方向性推荐，不过度承诺适配性
- 写CRM跟进记录必须先经销售确认后，再用CRM跟进记录助手写入
