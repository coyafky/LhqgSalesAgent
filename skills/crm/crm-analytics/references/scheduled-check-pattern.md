# 定时检查模式（Cron Hook Pattern）

> 用于 CRM 自动预警和提醒场景。非用户主动调用，由 cronjob 定时触发。

## 适用场景

| Hook | 检查条件 | 执行频率 |
|------|---------|:--------:|
| 沉睡线索预警 | 线索 >7/14/30天无跟进 | 每日一次 |
| 客户回访提醒 | 工单完工后3/7/30天 | 每日一次 |

## 通用结构

每个定时 hook skill 遵循以下模式：

### 1. 数据查询

使用 `+data-query` 或 `+record-list` 从 CRM 表中过滤出符合条件的记录。

```python
# 伪代码模式
records = query_table(
    table_id="tblXXX",
    filter={"状态条件": "xxx", "时间条件": "xxx"}
)
```

### 2. 条件匹配

对每条记录计算时间差，匹配预警节点：

| Hook | 节点 | 计算方式 |
|------|:----:|---------|
| 沉睡预警 | 7/14/30天 | 当前日期 - 最后跟进日期 |
| 回访提醒 | 3/7/30天 | 当前日期 - 实际完工时间 |

### 3. 查重（幂等性）

防止同一天重复推送同一条提醒。两种实现方式：

- **方案 A（推荐）**：在脚本中维护一份已提醒记录 `{记录ID}_{天数}` 的列表，每日检查
- **方案 B**：在 CRM 表中添加「最近回访日期」字段，写入后作为已提醒标记

### 4. 分级推送

| 级别 | 推送对象 | 推送内容 |
|:----:|---------|---------|
| 🔴 紧急 | 店长 | 预警简报 + 建议动作 |
| 🟠 中 | 店长 + 销售 | 预警简报 |
| 🟡 普通 | 销售 | 提醒 + 话术建议 |

### 5. 定时执行

```bash
# 使用 cronjob 设置每日定时任务
cronjob action=create \
  schedule="HH MM * * *" \
  name="任务名称" \
  prompt="执行{{任务描述}}，查询数据，生成报告并推送" \
  skills="crm-analytics" \
  deliver="origin"
```

两个 hook 建议错开执行时间：
- 客户回访提醒：`0 9 * * *`（早上9:00）
- 沉睡线索预警：`30 9 * * *`（早上9:30）

## 坑点

- **日期跨月**：跨月时 `CurrentMonth` 会切换，上月数据需用 `LastMonth`
- **时区**：cronjob 默认使用服务器时区，确认与 CRM 表时区一致
- **重复推送**：务必实现幂等性检查，避免销售每天早上收到同一条提醒
- **数据空洞**：如果表中有空字段（如无实际完工时间），跳过该记录，不要发送错误预警
