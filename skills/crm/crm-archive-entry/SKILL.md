# CRM客户档案助手

## 概述

客户成交后，销售用一句话描述客户车辆信息，助理自动搜索报价记录，写入「客户车辆档案表」并关联报价和跟进记录，完成客户归档。

- `../references/base.md` — Base token、身份、日期格式、确认边界。
- `../references/schema.md` — 客户档案、报价客户档案反向关联、销售人员字段格式。
- `../references/idempotency.md` — 建档前查重，避免同一客户车辆重复归档。
- `../references/people-products.md` — 销售负责人 open_id 与销售人员表 record_id。

## 工作流程

### 第一步：搜索客户

按客户姓名在以下表中搜索，获取已有信息：

1. **线索管理** — 获取来源、车型
2. **报价表** — 获取报价项目、成交金额（报价状态=已成交时优先）

### 第二步：解析车辆信息

从自然语言中提取：

- 车辆品牌+型号 → 客户基本信息
- 颜色 → 车辆颜色
- 年份+用途 → 客户基本信息
- VIN码 → 车架号VIN
- 车牌号 → 客户基本信息（客户车辆档案表无单独车牌号字段，记入基本信息）
- 客户状态关键词 → 客户状态
- 当前进展 → 一句话进展

### 第三步：追问缺失（最多3个）

按优先级：
1. 车辆基本信息（什么车？年份？）
2. 车辆颜色
3. 客户状态

### 第四步：确认摘要

```markdown
📋 准备写入 CRM — 客户车辆档案

| 字段 | 值 |
|------|-----|
| 客户名称 | XXX |
| 客户状态 | XXX |
| 客户基本信息 | XXX |
| 车辆颜色 | XXX |
| 一句话进展 | XXX |
| 销售负责人 | XXX |
| 客户标签 | XXX |

请回复「确认写入」后我写入 CRM。
```

### 第五步：写入

写入前按 `../references/idempotency.md` 搜索已有档案。同一客户同一车辆已存在时，更新指定档案；否则新建档案。必须保留新档案 `record_id`，用于回写报价表「客户档案」字段。

```bash
lark-cli base +record-upsert \
  --base-token Q6VQbuLTOaPP5VspfKXc7ZbTnA8 \
  --table-id tblV6qrgb7Xef1N7 \
  --json '{
    "客户名称":"王先生",
    "客户状态":"合作客户",
    "客户基本信息":"问界M9，2025年上牌，家庭用车",
    "车辆颜色":"珍珠白",
    "车架号VIN":"LSVAU2A36N2100001",
    "一句话进展":"刚贴完青龙YM-65车衣，满意度高",
    "客户标签":"重要客户",
    "销售负责人":[{"id":"ou_1c91ab71d9b6714030442cb1c6b85457"}],
    "销售人员表":[{"id":"recERZnykdciRR"}]
  }' \
  --as bot
```

### 第六步：关联报价表

创建档案后，如果有对应的已成交报价，更新报价表的「客户档案」字段：

```bash
lark-cli base +record-upsert \
  --base-token Q6VQbuLTOaPP5VspfKXc7ZbTnA8 \
  --table-id tblzGNVSFPSXgb9K \
  --record-id <报价记录ID> \
  --json '{"客户档案":[{"id":"<新档案记录ID>"}]}' \
  --as bot
```

### 第七步：验证回读

```bash
lark-cli base +record-get \
  --base-token Q6VQbuLTOaPP5VspfKXc7ZbTnA8 \
  --table-id tblV6qrgb7Xef1N7 \
  --record-id <新记录ID> \
  --as bot
```

## 安全规则

- 遵守 `../references/base.md` 的确认边界和隐私规则。
- 遵守 `../references/idempotency.md`：先查重再建档，不重复创建同一客户车辆档案。
- 写入前必须找到对应的成交报价或线索，无法确认时先让销售指定。
- 没有销售确认，不直接写入。
- 不凭空编造车辆信息。
- VIN码、车牌号等敏感信息只写 CRM 表，不写 workspace 文档。
