# CRM客户回访提醒Hook

## 概述

这是一个**定时执行的售后关怀 hook**。工单关闭后，按 3 天、7 天、30 天三个时间节点，自动提醒销售回访客户，确保售后关怀不漏掉，提升客户满意度和复购率。

## 共享引用

执行前按需读取：

- `../references/base.md` — Base token、身份、时区、隐私边界。
- `../references/schema.md` — 工单、客户档案、销售人员、产品字段。
- `../references/idempotency.md` — `workorder record_id + reminder day` 防重复规则。
- `../references/people-products.md` — 销售、店长、产品推荐解析。

**触发方式：** 通过 `cronjob` 设置每日定时任务
**目标：** 提升客户体验 + 增加复购/转介绍机会

## 回访节点设计

| 时间节点 | 回访目的 | 建议话术方向 | 提醒对象 |
|---------|---------|-------------|---------|
| **🎯 第3天** | 施工后关键关怀 | 提醒回店复检、3天不要洗车、检查有无问题 | 销售 |
| **🎯 第7天** | 使用感受跟进 | 问使用情况、满意度、有无异常 | 销售 |
| **🎯 第30天** | 长期维护关怀 | 邀约保养/养护、新品推荐、收集转介绍 | 销售 + 店长 |

## 规则明细

### 第3天提醒 — "刚完工关怀"

```
条件：工单已关闭 且 实际完工时间 = 3天前
状态：未发送过"第3天"提醒
目的：
  - 改色膜/车衣 → 提醒回店复检（3天复检期）
  - 窗膜 → 提醒 3-7 天不要升降车窗
  - 电动踏板 → 问安装后使用是否正常
```

### 第7天提醒 — "使用回访"

```
条件：工单已关闭 且 实际完工时间 = 7天前
状态：未发送过"第7天"提醒
目的：
  - 问使用感受
  - 确认有无质量问题
  - 收集满意度
  - 铺垫售后入口
```

### 第30天提醒 — "维护邀约"

```
条件：工单已关闭 且 实际完工时间 = 30天前
状态：未发送过"第30天"提醒
目的：
  - 邀约到店做养护/检查
  - 推荐关联产品（贴了车衣→推荐窗膜/改色）
  - 询问转介绍机会
  - 老客专属优惠提醒
```

## 执行流程

### 第一步：查询已关闭工单

查询工单表中符合以下条件的记录：
- 工单状态 = "已关闭"
- 实际完工时间不为空
- 实际完工时间在可追溯范围内（如最近60天内）

### 第二步：计算时间差

对于每条工单，计算：

```
天数 = 今天日期 - 实际完工时间
```

### 第三步：匹配回访节点

| 天数 | 节点 |
|:----:|------|
| 3 | 第3天回访 |
| 7 | 第7天回访 |
| 30 | 第30天回访 |

### 第四步：查重（防止重复提醒）

为避免每天重复推送，系统应记录**已发送提醒的工单+节点组合**，已发过的不再重复推送。

> 实现方式：在 skill 脚本中维护一个简单的已提醒记录，或通过 CRM 表中添加「最近回访日期」字段来标记。

## 脚本检查方案

```python
# 伪代码逻辑
from datetime import datetime, date

REMINDER_DAYS = [3, 7, 30]  # 回访节点

def check_revisit_reminders():
    today = date.today()

    # 1. 获取已关闭的工单
    orders = get_records("工单表", filter={
        "工单状态": "已关闭",
        "实际完工时间": {"不等于空": True}
    })

    for order in orders:
        completion_date = parse_date(order["实际完工时间"])
        days_since = (today - completion_date).days

        # 2. 匹配回访节点
        if days_since in REMINDER_DAYS:
            # 3. 查重：是否已提醒过
            if already_reminded(order["record_id"], days_since):
                continue

            # 4. 获取客户和销售信息
            customer = get_related_record("客户档案", order["客户"])
            salesperson = get_related_record("销售人员表", order["销售负责人"])

            # 5. 生成提醒
            reminder = generate_reminder(
                days=days_since,
                customer=customer,
                product=order["施工项目"],
                salesperson=salesperson
            )

            reminders.append(reminder)

    return format_reminder_report(reminders)
```

```bash

cronjob action=create \
  schedule="0 9 * * *" \
  name="客户回访提醒" \
  prompt="执行CRM客户回访提醒检查，查询3天/7天/30天前完工的工单，生成回访提醒并推送" \

  deliver="origin"
```

## 防重复机制

为防止同一天多次推送同一条提醒：

1. **已提醒标记**：在 skill 脚本中维护一个 `已提醒记录列表`，记录 `{工单ID}_{天数}` 的组合
2. **幂等检查**：每次执行前先检查是否已有该组合的记录
3. **历史清理**：超过 60 天的已提醒记录可自动清理
