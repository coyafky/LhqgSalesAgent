# CRM工单助手

## 概述

### 只读字段

工单编号（自动编号）、成交金额（lookup）、近7天判断（公式）、施工前照片/完工照片（附件，走专用命令）

### 工单状态流转

```
待安排 → 待施工 → 施工中 → 待验收 → 待交付 → 已关闭
                                                      ↘ 已取消（任意阶段可取消）
```

### 工单状态映射

| 销售说 | 映射为 | 说明 |
|--------|--------|------|
| 新工单/下个单/派工 | 待安排 | 刚创建，还没安排师傅 |
| 安排了师傅/排期了/明天做 | 待施工 | 已指定师傅和施工时间 |
| 开始施工/在贴了/正在做 | 施工中 | 师傅开始干活 |
| 做完了/贴好了/验收一下 | 待验收 | 施工完成，等验收 |
| 验收过了/没问题 | 待交付 | 验收通过，等客户提车 |
| 交付了/客户开走了/完工 | 已关闭 | 客户已提车，工单结束 |
| 取消/不做了/退单 | 已取消 | 工单取消 |

## 工作流程

### 第一步：搜索客户

1. 在**客户车辆档案表**中搜索客户：`+record-search --search-field fld1lpoKfI --keyword "王" --as bot`
2. 获取客户档案 record_id
3. 同时搜索**报价表**获取已成交的报价记录

### 第二步：匹配产品和师傅

1. 从销售说的话中提取产品名，在产品项目表中匹配
2. 从销售说的话中提取师傅名，匹配施工师傅

### 第三步：解析工单信息

- 时间 → 预约施工时间
- 师傅 → 施工师傅
- 状态关键词 → 工单状态

### 第四步：追问缺失（最多3个）

1. 施工项目（做什么产品？）
2. 施工师傅（谁来做？）
3. 预约施工时间（什么时候做？）

### 第五步：确认摘要

```markdown

| 字段 | 值 |
|------|-----|
| 客户 | XXX（关联档案：recXXX） |
| 来源报价 | XXX（报价编号） |
| 施工项目 | XXX |
| 施工师傅 | XXX |
```

### 第六步：写入

先按 `../references/idempotency.md` 判断是新建工单还是更新已有工单状态。状态更新必须定位到明确工单 `record_id`；新建工单需避免同一客户、报价、项目、预约时间重复创建。

## 安全规则

- 遵守 `../references/base.md` 的日期和确认边界。
- 遵守 `../references/idempotency.md`：状态更新必须绑定明确工单。
- 下工单前必须已有客户档案和已成交报价，无法确认时先让销售指定。
- 没有销售确认，不直接写入。
- 不凭空编造施工师傅和时间。
- 工单状态按实际进展更新，不可跳状态（如从"待安排"直接跳到"已关闭"）。
