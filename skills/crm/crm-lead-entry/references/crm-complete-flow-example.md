# CRM 全流程示例 — 吕先生

> 展示从线索录入到工单施工的完整 CRM 链路

## 背景

吕佳豪在抖音上接到一位吕先生，咨询问界M8电动踏板。以下是完整的跟进记录。

## 全链路逐步骤

### ① 线索录入

```
销售说：线索录入 吕佳豪 吕先生 佛山大良 问界M8 在微信上沟通了 电动踏板
助理追问：电话？来源？
销售补：电话 17628889999 来源是抖音

写入 → 线索管理表
- 客户姓名：吕先生
- 电话：17628889999
- 来源渠道：抖音
- 销售负责人：吕佳豪（rec6gBu3IVdtim）
- 车辆品牌：问界
- 车型：SUV（问界M8）
- 意向产品类型：轻改装（电动踏板）
- 线索状态：已分配
- 线索描述：问界M8，咨询电动踏板，佛山大良，微信沟通
```

### ② 跟进记录 + 预约到店

```
销售说：给这个吕先生的话，写一张跟进记录。也就是说他预计在后天的话到店，后想看一下电动踏板的情况，然后再安装。
助理确认 → 写入

写入 → 跟进记录表
- 客户名称：吕先生
- 跟进结果：已预约到店
- 跟进方式：微信沟通
- 跟进总结：预计后天（6月17日）到店看电动踏板，满意后安装
- 关联线索：recvmzXimrGz9y

同步更新 → 线索管理表
- 预计到店时间：2026-06-17
```

**关键技巧：** 当跟进结果=「已预约到店」且客户给了具体时间，要同步更新线索管理的「预计到店时间」字段，保持数据一致性。

### ③ 报价成交

```
销售说：我们在报价了 电动踏板的 MOD-EP-001 1980元 给他 且客户同意了这个报价
→ 自动查找产品 MOD-EP-001（recn4fuZRHHcQf）
→ 自动查找线索（recvmzXimrGz9y）

写入 → 报价表
- 客户姓名：吕先生
- 手机号：17628889999
- 车辆品牌：问界
- 车辆型号：问界M8
- 报价金额：¥1,980
- 报价方案：电动踏板 MOD-EP-001（不带灯），含安装，质保1年
- 报价状态：已成交
- 来源线索：recvmzXimrGz9y
- 报价人：吕佳豪（rec6gBu3IVdtim）

同步更新 → 线索管理表
- 线索状态：已转商机
```

### ④ 客户建档

```
销售说：报价成功，之后就可以直接下工单-待施工状态
注意：必须先有客户档案才能下工单，所以自动先建档案

写入 → 客户车辆档案表
- 客户名称：吕先生
- 客户状态：合作客户
- 客户基本信息：问界M8，安装电动踏板
- 一句话进展：已成交电动踏板MOD-EP-001，待后天到店安装
- 销售负责人：吕佳豪

同步更新 → 报价表
- 客户档案：关联到新创建的档案记录
```

### ⑤ 下工单 + 指定师傅

```
自动创建 → 工单表（初始状态：待安排）
- 客户：关联档案
- 来源报价：关联报价
- 施工项目：MOD-EP-001
- 预约施工时间：2026-06-17
- 销售负责人：吕佳豪

销售补：梁增炯

更新 → 工单表
- 施工师傅：梁增炯
- 工单状态：待施工
```

### ⑥ 完工（后续）

```
销售说：吕先生完工了 → 工单状态→已关闭
（后续自动触发CRM客户回访提醒Hook，在第3/7/30天提醒回访）
```

## 核心经验

| 环节 | 技巧 |
|------|------|
| 查重 | 写线索前先搜电话+姓名，与吕佳豪的另一位王先生区分开 |
| 双写 | 跟进记录写「已预约到店」时，同步更新线索的「预计到店时间」 |
| 报价成交 | 客户同意=已成交，同时更新线索状态为「已转商机」 |
| 先档后单 | 下工单前必须已有客户档案，没有的话要自动创建 |
| 师傅确认 | 工单初始为「待安排」，指定师傅后改为「待施工」 |
