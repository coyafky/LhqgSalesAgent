## 工作流程

### 第一步：解析

从销售的自然语言中提取：

- 客户称呼 → 客户姓名
- 手机号 → 电话
- 来源关键词 → 来源渠道
- 销售名 → 销售负责人（查映射表获得 record_id）
- 车型 → 车型（单选）+ 车辆品牌/线索描述
- 产品意向 → 意向产品类型
- 线索状态 → 指定了销售则「已分配」，未指定则「未分配」

### 第二步：追问缺失（最多3个）

按优先级：
1. 电话
2. 来源渠道
3. 下次跟进时间 / 预算范围

### 第三步：查重

写入前按以下顺序搜索重复线索：
1. 电话精确匹配
2. 客户姓名 + 车型

发现重复时，让销售选择「更新现有」或「新建一条」。

### 第四步：确认摘要

```markdown
📋 准备写入 CRM — 线索管理

| 字段 | 值 |
|------|-----|
| 客户姓名 | XXX |
| 电话 | XXX |
| 来源渠道 | XX |
| 销售负责人 | XX |
| 车辆品牌 | XX |
| 车型 | XX |
| 意向产品 | XX |
| 线索状态 | 已分配 |
| 线索描述 | XX |

请回复「确认写入」后我写入 CRM。
```

### 第五步：写入

使用 `+record-upsert` 命令写入。写入前先搜索重复线索（按电话→客户名+车型）。

没有重复时创建新记录；发现重复时，经销售确认后更新指定记录。

⚠️ **关键写规则：**
- 销售负责人 = `[{"id": "rec6gBu3IVdtim"}]`（关联字段，用销售人员表 record_id）
- 客户姓名 和 姓名 两个字段都填
- 意向产品类型 = `["轻改装"]`（多选用数组）
- 车型 = `"SUV"`（单选，用选项名）

### 第六步：验证回读

写入后用 `+record-get` 回读确认，返回给用户。

## 安全规则

- 没有销售确认，不直接写入 CRM。
- 不凭空补手机号、预算、成交金额。
- 遇到重复客户，让销售选择操作哪一条。
- 不把推测信息写成事实，推测内容放「线索描述」。
- 手机号等隐私只进入 CRM，不写入 workspace 文档。
- 价格、折扣、活动政策必须由销售明确提供后才写入。
